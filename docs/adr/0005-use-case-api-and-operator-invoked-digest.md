# ADR-0005: API сценариев и ручной запуск дайджеста

## Статус

Принято

## Контекст

На старте продукт должен решать реальные задачи пользователя, а не отражать внутреннюю структуру таблиц. Одновременно автоматический дайджест по расписанию добавляет фоновую хрупкость до того, как понятен устойчивый сценарий использования.

## Решение

API проектируется вокруг сценариев использования, а не вокруг CRUD-сущностей. Дайджест в MVP запускается вручную как `operator-invoked workflow` из CLI и доставляется в Telegram Bot только по запросу.

## Последствия

- API лучше соответствует реальному JTBD.
- Внутреннюю схему БД можно менять без разрушения пользовательских контрактов.
- Уменьшается риск ложной автоматизации вокруг дайджеста.
- Автоматический дайджест по расписанию остаётся допустимым будущим расширением после стабилизации сценария использования.
