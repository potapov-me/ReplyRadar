# ADR-0002: PostgreSQL как источник истины

## Статус

Принято

## Контекст

Система хранит raw messages, derived facts, summaries и embeddings. Для MVP критично сохранить простую и проверяемую модель консистентности.

## Решение

`PostgreSQL` выбран как единственный источник истины. Очередь используется только как транспорт для realtime-triggered обработки. `pgvector` хранится рядом с транзакционными данными в том же Postgres.

## Последствия

- Консистентность и reprocessing проще контролировать.
- Нет отдельного operational контура под message broker или vector DB.
- SQL остаётся основным инструментом отладки и анализа.
- При росте нагрузки возможен будущий вынос части функций в отдельные сервисы, но не требуется для MVP.
