# ADR-0004: Локальный LLM через LM Studio и LiteLLM

## Статус

Принято

## Контекст

ReplyRadar обрабатывает личные переписки. На раннем этапе приватность данных важнее, чем максимальная модельная мощность. Нужен также стабильный программный интерфейс, чтобы не привязывать код к одному провайдеру SDK.

## Решение

Все LLM-вызовы в MVP идут локально через `LM Studio`, который предоставляет OpenAI-совместимый API. `LiteLLM` используется как унифицированный клиентский слой.

## Последствия

- Данные не покидают локальный контур по умолчанию.
- Снижается зависимость от внешних API и сетевой доступности.
- Качество моделей и скорость ограничены локальной средой пользователя.
- Смена модели и возможная поддержка других backends остаются конфигурационной задачей.
