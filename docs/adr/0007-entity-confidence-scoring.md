# ADR-0007: Система уверенности для фактов и связей

## Статус

Принято

## Контекст

Факты и связи в Entity Knowledge Graph извлекаются LLM из неструктурированного текста и могут быть неточными, неполными или противоречивыми. Часть фактов говорит сам человек о себе, часть — третьи лица. Часть фактов устаревает со временем. Без явной оценки уверенности система будет смешивать достоверные и спекулятивные утверждения.

## Решение

Каждый факт и каждая связь хранят четыре компонента уверенности:

- `base_confidence` (float, 0.0–1.0) — оценка LLM при извлечении
- `corroboration_count` (int) — число независимых сообщений, подтверждающих факт
- `contradiction_count` (int) — число сообщений, противоречащих факту
- `source_type` ('self' | 'other' | 'inferred') — кто является источником

Финальная уверенность вычисляется при запросе по формуле:

```
score = base_confidence
      × source_weight          # self=1.0, other=0.7, inferred=0.5
      × staleness_decay         # exp(-0.693 × days / 365), период полураспада 1 год
      × corroboration_factor    # 1.0 + min(0.4, count × 0.1)
      × contradiction_penalty   # 1.0 / (1.0 + count × 0.5)
```

Пороги при выдаче:
- `≥ 0.75` — достоверный факт, показывается без оговорок
- `0.40–0.75` — показывается с пометкой "вероятно"
- `< 0.40` — скрывается по умолчанию, доступен при явном запросе

Корробобрация считается по таблице источников (`entity_fact_sources`, `entity_relation_sources`) с уникальным ключом `(fact_id, message_id)` — исключает двойной счёт из одного треда.

При транзитивных запросах уверенность по цепочке перемножается. Это делает глубокие связи естественно менее уверенными без дополнительной логики.

## Последствия

- Формула не хранится в БД и может меняться без миграций.
- Компоненты уверенности хранятся отдельно — можно анализировать причины низкой оценки.
- Противоречащие факты не удаляются: оба хранятся с пониженной уверенностью, старый помечается `superseded_by`.
- Калибровка порогов (0.75 / 0.40) предварительная; уточняется на реальных данных в `evals/`.
