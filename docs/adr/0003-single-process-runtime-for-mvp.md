# ADR-0003: Единый процесс приложения для MVP

## Статус

Принято

## Контекст

ReplyRadar объединяет listener, processing engine, API, bot и scheduler. Для ранней стадии важно минимизировать число подвижных частей.

## Решение

Для MVP система работает как один процесс приложения с общим `asyncio` event loop. Realtime ingestion, backfill, обновление резюме и API разделяются логически, но не разворачиваются как отдельные исполняемые сервисы.

## Последствия

- Ниже операционная стоимость и проще локальный запуск.
- Выше риск shared-failure-domain по сравнению с раздельными процессами.
- Этот риск принят осознанно, потому что продукт single-user и имеет явную модель деградации.
- Первое post-MVP усиление: durable ingest buffer, а не полный переход на распределённый runtime.
