# Документация

## Структура

- [Архитектура](./architecture.md) — текущая целевая архитектура MVP
- [Разбор](./review.md) — архитектурный разбор и остаточные риски
- [Карта пользовательского пути](./cjm.md) — ключевые сценарии использования
- [ADR](./adr/README.md) — архитектурные решения в формате ADR

## Принцип ведения документации

- `README.md` в корне описывает проект и текущий статус
- `docs/architecture.md` хранит актуальную целевую архитектуру
- `docs/adr/` фиксирует устойчивые решения, которые не должны теряться в обсуждениях
- CJM и review живут рядом с архитектурой, чтобы продуктовые и технические решения были видны в одном месте
