## Правило актуальности

- `README.md` в корне фиксирует то, что реально можно запустить сейчас
- обзорные файлы в `docs/` не должны ссылаться на несуществующие эндпоинты как на готовые
- future-facing документы должны явно помечать, что описывают roadmap, а не текущий runtime
